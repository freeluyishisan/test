# 东八区天气交易策略

## 核心判断

东八区天气盘口赚钱不靠泛泛天气预报，靠四件事：

```text
结算站点准确
官方数据更新快
盘中 nowcasting 准
盘口价格没有完全反应
```

## 市场分类

### 1. 城市官方站最高温

代表：香港 HKO。

特点：

```text
站点明确
分钟级或 10 分钟数据重要
雷达、云图、辐射、风向影响大
盘中可预测性强
```

### 2. 机场最高温 / METAR 站点

代表：RKSI、RJTT、北京机场、深圳机场。

特点：

```text
METAR 多为整点/半点/特殊报
温度常取整数
盘口可能被机场突发风向影响
海风/跑道附近环境重要
```

### 3. 城市区域盘口

代表：台北、首尔市区类盘口。

特点：

```text
必须先确认结算源
不能混用城市平均
官方结算规则比模型更重要
```

## HKO 第一版策略

### 时间段权重

| 时间 | 重点 |
|---|---|
| 00:00–06:30 | 夜间底温、湿度、云雨背景 |
| 07:00–10:30 | 升温启动速度 |
| 11:00–12:30 | 当天基础上限判断 |
| 13:30–15:15 | 最高温决胜窗口 |
| 15:30 后 | 是否还有尾盘刷新高 |

### HKO 决策主线

```text
当前温度
+ 今日已录得最高温
+ 近 10/30/60 分钟升温斜率
+ King's Park 辐射
+ 雷达雨区
+ 卫星云图
+ 风向风速
= 阈值突破概率
```

### 阈值概率

每个盘口都转成概率：

```text
P(最高温 >= 30°C)
P(最高温 >= 31°C)
P(最高温 >= 32°C)
P(最高温 >= 33°C)
```

### 入场逻辑

```text
模型概率 - 盘口价格 >= 8%
且数据新鲜
且盘口深度够
且不处于数据异常状态
才进入候选
```

### 不交易场景

```text
数据超过 3 分钟未更新
站点不是 HKO
今日最高字段缺失
雷达/云图无法读取且临近关键时段
盘口价格已经充分反应
订单簿深度太薄
```

## 机场站策略

机场市场不能照搬 HKO。

重点因素：

```text
METAR 温度是否整数结算
跑道附近风向
海陆风切换
TAF 趋势
临近站对比
太阳辐射/云量
雷暴/阵雨特殊报
```

机场盘口优先建模：

```text
当前 METAR 温度
最近 3–6 小时 METAR 变化
TAF 关键词
官方小时预报
数值预报
雷达/云图
```

## 赚钱条件

```text
真实概率判断 > 市场隐含概率 + 交易成本 + 安全边际
```

不满足就不交易。

## 第一版保守参数

```text
最小 edge：8%
强信号 edge：12%
单笔最大模拟仓位：20 USDC
单日最大模拟亏损：100 USDC
自动交易：关闭
```
